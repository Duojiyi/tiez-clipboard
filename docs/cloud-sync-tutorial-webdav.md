# WebDAV 云同步配置教程

本教程介绍如何在 Magpie 中配置 WebDAV，实现剪贴板历史在多台设备间的云端同步与备份。

> WebDAV 云同步面向「历史存档与跨设备同步」场景：通过你自己的 WebDAV 存储空间定期上传 / 拉取剪贴板数据。需要实时推送请改用 MQTT 同步（见 `cloud-sync-tutorial-mqtt.md`）。
>
> 该功能当前处于 Beta 阶段。

## 一、准备一个 WebDAV 服务

你需要一个支持 WebDAV 协议的存储服务。常见选择：

- **Nextcloud / ownCloud**：自建或托管，WebDAV 地址形如 `https://your-domain/remote.php/dav/files/用户名`。
- **坚果云**：在「账户信息 → 安全选项」中开启第三方应用并生成应用密码，使用其提供的 WebDAV 地址。
- **群晖 NAS（Synology）**：开启 WebDAV Server 套件后通过 `https://NAS地址:端口` 访问。
- **其他兼容 WebDAV 的网盘 / 服务器**。

记录下以下信息：

- WebDAV 服务地址（URL）
- 用户名
- 密码（部分服务需使用「应用专用密码」而非登录密码）

## 二、在 Magpie 中打开云同步设置

1. 打开 Magpie 主面板，进入「设置」。
2. 找到「云端同步」分组（标题旁标有 Beta）并展开。
3. 打开「启用云端同步」开关。

## 三、填写 WebDAV 连接参数

| 配置项 | 说明 |
| --- | --- |
| **WebDAV 地址（URL）** | WebDAV 服务的完整地址，例如 `https://dav.example.com/remote.php/dav/files/user`。建议使用 `https` 以保证传输加密。 |
| **用户名** | WebDAV 账户用户名。 |
| **密码** | WebDAV 账户密码或应用专用密码。坚果云等服务请使用生成的应用密码。 |
| **同步目录（Base Path）** | 数据在 WebDAV 上存放的目录，默认为 `tiez-sync`。多台设备需填写相同目录才能互相同步。 |

> 填写完地址、用户名、密码后，可点击下方的「立即同步」验证配置是否正确。

## 四、配置同步行为

| 配置项 | 说明 |
| --- | --- |
| **自动同步** | 开启后按设定的间隔自动上传 / 拉取，无需手动操作。 |
| **同步间隔（秒）** | 自动同步的轮询间隔，范围 5~3600 秒，默认 120 秒。 |
| **快照间隔（分钟）** | 生成完整数据快照的间隔，范围 5~1440 分钟，默认 720 分钟。 |

## 五、选择同步内容范围

在「同步内容范围」中可以分别控制哪些类型的内容参与同步：

- **文本**：同步纯文本与富文本内容。
- **图片**：同步图片内容（数据量较大，按需开启）。
- **文件路径**：同步复制的文件路径记录。
- **表情包**：同步表情包内容。

按需勾选即可，关闭的类型不会上传到云端。

## 六、查看同步状态

「云端同步」分组内提供状态面板，显示：

- 当前状态：空闲（idle）/ 同步中（syncing）/ 错误（error）/ 未启用。
- 已上传 / 已接收的条目数。
- 最近一次同步时间。
- 若出现错误，会显示最近一次的错误信息。

你也可以随时点击「立即同步」手动触发一次同步。

## 七、多设备同步要点

1. 每台设备都连接到**同一个 WebDAV 服务**，使用相同账户。
2. 每台设备的**同步目录（Base Path）保持一致**。
3. 各台设备都开启「启用云端同步」。

## 八、隐私与安全建议

- 务必使用 `https` 地址，避免账号与内容明文传输。
- 优先使用服务方提供的「应用专用密码」，不要直接填登录主密码。
- 图片等大体积内容按需同步，以节省云端空间与带宽。

## 常见问题

**Q：点击「立即同步」报错？**
检查 WebDAV 地址是否完整且可访问、用户名密码是否正确、部分服务是否需要应用专用密码、地址是否使用了 `https`。

**Q：两台设备数据对不上？**
确认两台设备使用相同的 WebDAV 账户与相同的同步目录（Base Path），并都已开启云端同步。

**Q：同步会很耗流量吗？**
文本同步数据量很小。若开启图片同步，数据量会明显增大，可在「同步内容范围」中关闭图片以节省流量。
