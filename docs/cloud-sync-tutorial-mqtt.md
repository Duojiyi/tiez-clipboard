# MQTT 实时同步配置教程

本教程介绍如何在 Magpie 中配置 MQTT，实现多台设备之间的剪贴板实时同步。

> MQTT 同步面向「实时推送」场景：在一台设备复制内容后，另一台已连接同一 MQTT 服务器、同一主题的设备会近乎即时地收到。它不负责历史存档，历史存档请使用 WebDAV 云同步（见 `cloud-sync-tutorial-webdav.md`）。

## 一、准备一个 MQTT 服务器

你需要一个可用的 MQTT Broker。常见选择：

- **公共测试 Broker**（仅用于体验，请勿传输隐私内容）：例如 `broker.emqx.io`。
- **自建 Broker**：在自己的服务器上部署 [EMQX](https://www.emqx.io/) 或 [Mosquitto](https://mosquitto.org/)。
- **云托管 MQTT 服务**：使用 EMQX Cloud 等托管服务。

记录下以下信息，配置时需要用到：

- 服务器地址（host），如 `mqtt.example.com`
- 端口（port）
- 用户名 / 密码（如果 Broker 开启了认证）
- 连接协议（普通 TCP、SSL/TLS、WebSocket 等）

## 二、在 Magpie 中打开 MQTT 同步设置

1. 打开 Magpie 主面板，进入「设置」。
2. 找到「同步设置」分组并展开。
3. 打开「启用同步」开关，下方会显示完整的 MQTT 配置项。

## 三、填写连接参数

| 配置项 | 说明 |
| --- | --- |
| **协议（Protocol）** | 选择连接方式。`mqtt://` 为标准 TCP（默认端口 1883）；`mqtts://` 为 SSL/TLS 加密（8883）；`ws://` 为 WebSocket（8083）；`wss://` 为加密 WebSocket（8084）。 |
| **WS 路径（WS Path）** | 仅当协议为 `ws://` 或 `wss://` 时出现，填写 WebSocket 路径，通常为 `/mqtt`。 |
| **服务器地址（Host）** | MQTT Broker 的域名或 IP，例如 `mqtt.example.com`。不要带协议前缀。 |
| **端口（Port）** | 切换协议时会自动填入对应默认端口，如与服务器实际端口不同请手动修改。 |
| **用户名（User）** | Broker 的登录用户名，未启用认证时可留空。 |
| **密码（Password）** | Broker 的登录密码，未启用认证时可留空。 |
| **主题（Topic）** | 用于同步的 MQTT 主题。**同步的多台设备必须填写完全相同的主题**，例如 `tiez/my_device`。 |
| **消息通知** | 开启后，收到其他设备同步过来的内容时会弹出系统通知。 |

> 提示：协议与端口要与服务器的实际监听方式匹配。例如开启了 TLS 的 Broker 必须选择 `mqtts://` 或 `wss://`，否则会连接失败。

## 四、确认连接状态

填写完成后，「同步设置」分组标题旁会显示一个状态圆点：

- 🟢 绿色：已连接（Connected）
- 🟠 橙色：连接中（Connecting）
- 🔴 红色：未连接 / 连接失败（Disconnected）

如果长时间为红色，请检查：服务器地址与端口是否正确、协议是否匹配、用户名密码是否正确、网络与防火墙是否放行对应端口。

> 修改连接参数后如未立即生效，可重启 Magpie 使配置重新加载。

## 五、多设备同步要点

1. 每台设备都按上述步骤连接到**同一个 MQTT 服务器**。
2. 每台设备都填写**完全相同的主题（Topic）**。
3. 各台设备都打开「启用同步」开关。

满足以上三点后，在任意一台设备复制内容，其他设备即可实时收到。

## 六、隐私与安全建议

- 优先使用 `mqtts://` 或 `wss://` 等加密协议，避免内容在网络中明文传输。
- 不要在公共测试 Broker 上传输密码、隐私等敏感内容。
- 为不同的同步圈子使用不同且不易猜测的主题名，避免内容被无关设备订阅到。

## 常见问题

**Q：连接一直显示红色？**
检查协议与端口是否匹配（如 TLS 端口配了非加密协议）、Broker 是否要求认证、网络是否能访问该服务器。

**Q：两台设备连上了但收不到对方内容？**
确认两台设备填写的主题（Topic）逐字一致，且都已开启「启用同步」。

**Q：MQTT 同步会保存历史记录吗？**
不会。MQTT 只做实时推送。需要跨设备保留历史，请配合使用 WebDAV 云同步。
