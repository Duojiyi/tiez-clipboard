# Magpie 18 个月进化路线

> 状态：草案（v0.4.0 发布日 2026-05-27 开始计算）
> 目标：从「剪贴板工具」演进为「Personal Information Hub」
> 衡量标准：v1.0 用户感觉「这是我每天必开的中央枢纽」，GitHub 几万 star，至少一个细分用户群把它当成首选

---

## 总览

```
2026-06 ─────── 2026-08 ─────── 2026-11 ─────── 2027-02 ─────── 2027-05 ─────── 2027-08 ─────── 2027-11
   │              │                │                │                │                │                │
   │ Q1 站稳     │ Q2 智能识别     │ Q3 知识资产     │ Q4 本地 AI       │ Q5 启动器整合    │ Q6 v1.0 + 出圈   │
   │ Stability   │ Smart Capture   │ Knowledge       │ AI Workspace    │ Quick Launcher   │ Hub Identity     │
   │ v0.4.1-0.4.3│ v0.5.0          │ v0.6.0          │ v0.7.0           │ v0.8.0           │ v1.0.0           │
   ▼              ▼                ▼                ▼                ▼                ▼                ▼
```

**节奏**：每 3 个月一个 minor 版本，每个版本有清晰主题和验收标准。
**基调**：根不丢（剪贴板基础体验持续稳定），渐进式加（每一步都让现有用户自然接受）。

## Q2 (2026-08 ~ 2026-11)：智能识别 + 一键操作

**版本**：v0.5.0
**主题**：Smart Capture
**用户感知**：「Magpie 知道我复制了什么，主动给我下一步」

### 核心改造

#### 1. **识别器（Recognizer）系统**
为每种内容类型加识别逻辑，复制即识别：

| 类型 | 识别 | 触发动作 |
|------|------|----------|
| URL | 协议头 + 域名格式 | 打开 / 翻译 / 转 Markdown / Obsidian |
| 邮箱 | RFC 简单匹配 | 复制 / 写邮件 / 加联系人 |
| 手机号 | 中外格式 | 拨打（手机端）/ 添加联系人 / 短信预览 |
| 身份证 | 中国 18 位 | OCR 字段 / 脱敏复制 |
| 银行卡 | 数位规则 | 脱敏 / 一键存档 |
| 地址 | 关键词（路/号/室）| 地图打开 |
| JSON | `{...}` 结构 | 格式化 / 转 YAML |
| SQL | 关键词 | 格式化 / 在 IDE 中打开 |
| 代码 | 语法启发式 | 高亮预览 / 在 ChatGPT 中讨论 |
| Markdown | `# / *` 等 | 渲染预览 / 复制 HTML |
| 颜色 | hex / rgb | 色卡预览 / 转其他格式 |
| 时间戳 | unix | 转可读时间 |
| 表格 | TSV / CSV | 渲染预览 / 转 Excel |
| 长文 | > 500 字 | 总结 / 翻译 / 朗读 |

#### 2. **Action 系统**
每个 Action 是一个独立模块，输入剪贴板内容、输出新内容或副作用。

#### 3. **唤起方式**
- 复制后自动短暂浮现「智能动作条」（可关闭）
- 主面板条目右键 → 智能动作菜单
- 全局快捷键唤起

### 不做的
- ❌ 不做联网识别（隐私优先）
- ❌ 不做 LLM 自动调用（用户主动触发才调）
- ❌ 不做 OCR（推迟到 Q4）

### 关键里程碑
- ✅ 至少 8 种识别器 + 20 种 Action
- ✅ 识别延迟 < 100ms（不影响复制体验）
- ✅ 用户能自定义启用/禁用每种识别
- ✅ 第三方插件接口（暴露 Recognizer / Action API）

## Q3 (2026-11 ~ 2027-02)：知识资产积累

**版本**：v0.6.0
**主题**：Knowledge Capture
**用户感知**：「Magpie 不只是工具，开始变成我的知识助手」

### 核心改造

#### 1. **自动主题聚类**
- 本地嵌入模型（Ollama 调 nomic-embed-text 或 bge-m3）
- 复制条目自动生成 embedding 存 SQLite (sqlite-vss)
- 周期性聚类，给出主题标签

#### 2. **智能回顾**
- 每周/每月汇总：「这周你重点关注了什么」
- 间隔重复（Spaced Repetition）：3 个月前复制过的精华内容定期推送

#### 3. **跨条目搜索 v2**
- 当前只能字面搜索
- 升级为语义搜索：「找我之前复制过的关于 React 性能的内容」

#### 4. **主题报告**
- 选中一个主题 → 生成 Markdown 整理
- 一键导出到 Obsidian / Notion / 本地文件

#### 5. **数据资产化**
- 用户能感知到「我有多少条复制历史，分布是什么样的」
- 仪表盘显示：常复制的应用、活跃时间、内容类型分布

### 关键里程碑
- ✅ 5000+ 条数据下，语义搜索 < 200ms
- ✅ 嵌入模型本地运行（不强制联网）
- ✅ 用户访谈：≥ 50% 觉得「这个回顾真的有用」

## Q4 (2027-02 ~ 2027-05)：本地 AI 工作台

**版本**：v0.7.0
**主题**：AI Workspace
**用户感知**：「Magpie 接 AI 的能力比单独打开 ChatGPT 还顺手」

### 核心改造

#### 1. **多 AI 后端支持**
- Ollama（本地，主推）
- LM Studio（本地）
- OpenAI API（通用）
- Claude / Gemini（可选）
- LinkAPI 等中国转发服务（实用）

#### 2. **Prompt 库**
- 内置 20+ 常用 Prompt（翻译/总结/改写/扩写/朗读/分类/挑错）
- 用户可自定义 + 分享 Prompt（社区库）

#### 3. **AI 操作层**
- 选中条目 → 应用 Prompt → 输出新内容
- 批量处理：选中多条 → 统一处理

#### 4. **OCR**
- 用 Windows.Media.Ocr（系统自带，免装模型）
- 图片右键 → OCR 提取文字

#### 5. **隐私保护**
- 所有 AI 调用前显示数据出境警告（可关闭）
- 提供「本地优先」总开关
- AI 历史本地存储，不上云

### 关键里程碑
- ✅ 至少 3 种 AI 后端可用
- ✅ Prompt 切换 < 100ms
- ✅ 隐私警告体验不烦人（一次性提示，可静音）
- ✅ Windows OCR 集成无需用户额外装东西

## Q5 (2027-05 ~ 2027-08)：启动器整合

**版本**：v0.8.0
**主题**：Quick Launcher
**用户感知**：「我桌面上一个工具就够了」

### 核心改造

把 Magpie 主面板升级为「全能启动器」，剪贴板成为它的一个 tab。

#### 1. **命令系统**
全局热键（与剪贴板分开）→ 弹小窗，输入命令：

| 命令前缀 | 功能 |
|----------|------|
| 无前缀 | 模糊搜索：剪贴板 / Snippet / 应用 |
| `>` | 执行命令（如 `>open 路径`） |
| `=` | 计算器（`=56*1.13`）|
| `tr` | 翻译（`tr hello world`）|
| `ai` | 一句问 AI |
| `web` | 在线搜索 |
| `app` | 启动应用 |

#### 2. **插件接口**
- 第三方写命令插件
- Magpie 的「智能动作」也是命令插件的特例

#### 3. **统一入口**
- 现在 Magpie 主面板（剪贴板）+ Snippet 面板 + 表情包面板
- 升级后：一个统一的「启动器面板」，用 tab / 模式切换

### 不做的
- ❌ 不试图取代 PowerToys Run / Raycast 完整功能
- ❌ 不做应用启动器主战场（让系统去做）
- ✅ 专注「围绕剪贴板/信息流转的快速操作」

### 关键里程碑
- ✅ 启动器响应 < 50ms
- ✅ 至少 8 种内置命令
- ✅ 第三方插件 API 文档完整
- ✅ 用户可以完全用启动器而不需要主剪贴板面板

## Q6 (2027-08 ~ 2027-11)：v1.0 + 出圈

**版本**：v1.0.0
**主题**：Personal Information Hub
**用户感知**：「这是我桌面上信息流转的中央枢纽」

### 核心改造

这一版**不再加大功能**，而是**重新定义自己是什么**。

#### 1. **Brand & Positioning**
- README 整体重写：「Magpie - Your personal information hub」
- 新 Logo（如有需要）
- 新 Tagline（强调「Information Hub」而不是「Clipboard Manager」）
- 主题截图全部用 v0.4.x 之后的产品

#### 2. **Onboarding 重做**
- 首次启动 5 分钟引导（视频 + 卡片）
- 内置教学：「试试复制一段代码看看」「试试复制一个 URL」
- 每个核心功能首次使用都有提示

#### 3. **文档站**
- docs.magpie.app（或类似）
- 用户手册中英双语
- 视频教程（B 站 + YouTube）
- API 文档

#### 4. **稳定性兜底**
- 24 小时持续不崩
- 100 万条历史完全不卡
- panic 兜底完善
- 所有错误有清晰中英对照

#### 5. **生态启动**
- Obsidian 插件 v1
- VSCode 插件 v1
- Raycast 命令包（macOS）
- 标准化 Magpie 协议（让其他工具能集成）

### 关键里程碑
- ✅ GitHub star 5k+
- ✅ 月活 5000+ 用户
- ✅ Show HN 一次成功上首页（已经做的话不算）
- ✅ Product Hunt 至少 #5 of the day
- ✅ 至少一个细分群体（程序员 / 学生 / 律师 等）公开把 Magpie 当推荐工具

## 横向贯穿的工程任务

下面是不绑定某个版本，但需要持续做的事：

### 性能基线
- 每个 minor 版本前测 5 个核心指标，写 perf-baseline.md
- 启动时间、复制响应、主面板打开延迟、搜索响应、内存占用

### CI / 工具链
- Q1 末完成 Rust cache（CI 时间 < 15min）
- Q2 加 e2e 测试（playwright）
- Q3 加 benchmark CI

### 国际化
- 主语言：中文（简）+ 英文
- 次语言：日语 / 繁中（社区翻译）
- 全部错误信息中英对照

### 跨设备同步
- Q1：保持现有 MQTT + WebDAV 稳定
- Q2：加端到端加密
- Q3：协议插件化（为未来 Android / iOS 客户端做准备）
- Q4-Q6：暂不投入移动端，等 Hub 站稳

### 数据兼容承诺
- v0.4.x 立下的内部标识符（tiez.log / TIEZ_RICH_IMAGE / tiez-sync 等）持续保留
- 等到 v1.0 minor bump 时一次性迁移并出迁移代码

## 风险与回应

| 风险 | 回应 |
|------|------|
| 用户不接受「Information Hub」定位 | 渐进式改造，每版本可独立工作；定位升级放 v1.0 |
| 智能识别误判频繁 | 默认全关，用户主动启用；提供反馈入口 |
| 本地 AI 推理硬件门槛 | 多 AI 后端可选，云端兜底 |
| 启动器引入复杂度 | 启动器和剪贴板独立可关，不强制 |
| 某季度做不完 | 砍非 P0，周期延 1-2 个月，不慌 |
| 跟 Raycast / Alfred 撞产品定位 | 强调「围绕剪贴板的信息中枢」差异 |
| 商业产品（Typeless/Paste）升级太快 | 不追，专注隐私 + 本地 + 中文优势 |
| 个人精力不够 | 每个版本只做能 part-time 完成的量 |

## 退出策略（如果中途想停）

无论何时停在哪一版，下一版都不应该「鸡肋化」：

| 停在 | 用户感觉 |
|------|----------|
| v0.4.3 | 「一个好用的剪贴板工具，没问题」 |
| v0.5.0 | 「比同类工具多了智能动作，挺方便」 |
| v0.6.0 | 「能积累知识，比单纯剪贴板更值得长期用」 |
| v0.7.0 | 「带 AI 的剪贴板工具，独此一家」 |
| v0.8.0 | 「Windows 上的轻量 Raycast 替代」 |
| v1.0 | 「桌面信息中枢」 |

每一版都是「可发布的完整产品」，不是「半成品」。

---

## 心态

- **不要被「打 Typeless / Raycast」的诱惑带偏**：定位差异化，不正面交锋
- **不要追求「立刻火」**：18 个月是长跑，月增长 10% 比月增长 100% 更可持续
- **不要忘记「我自己用得爽」**：dogfood 是最好的产品经理
- **不要把开源当成义务**：累了停一停，开源没有 deadline
- **不要忘记其他人生**：Magpie 是项目，不是你的全部
