# Magpie v0.4.1 计划：Stability + UX + UI 升级（合并原 0.4.2）

> 状态：v0.4.0 发布后定稿，2026-05-29 修订
> 主题：稳定性 / 性能内功 + 上游 issue 借鉴 + UX 增强 + UI 升级 + 内置表情包库
> 周期：v0.4.0 发布后 8 周（合并 0.4.2 后周期相应延长）
> 路线图归属：方案 A（迁移巩固）+ 方案 C（性能/稳定性专项）+ U/F 系列上游借鉴 + V/UI 升级

---

## 0. 修订说明（2026-05-29）

合并原计划与 0.4.2 计划。一度想砍掉 A7 / U5 / F5 / C7，复盘后**只砍 C7**，其余恢复。判断标准从「小 / 快 / 不偏题」修正为：**是否真实存在、是否有价值、风险是否可控**。用这把尺子量，只有 C7 命中「有真实下行且价值不足」。

- **C7（`opt-level = "z"`）确认砍掉**：桌面应用不该为体积牺牲性能，收益小、有真实下行。要瘦身用 `lto = "fat"` + `strip` 这类不损性能的手段
- **A7（Mac CSS 修）恢复 → P3**：成本极小（additive CSS）、对 Windows 零风险、能为未来 Mac 适配除雷；唯一代价是无法在本仓库验证 → 只做「明显正确」的样式兜底，不做复杂适配
- **U5（25H2 IME 吞字）恢复 → P1（限时调研）**：对中文用户 IME 兼容是**核心**不是边缘，「等反馈量」=等用户流失。技术不确定性高 → 用 **2~3 天 spike** 应对，而不是无限推迟
- **F5（快捷键 scope 分离）恢复 → P1**：本期主题已含 UX，scope 分离正是 UX；改造虽大但合并后周期足够。**默认全部标 `Global` = 与 v0.4.0 行为逐字节一致**，零回归
- **新增 A8 / A9 / A10 / C9**：迁移失败回滚、应用内反馈入口、便携版自启动 bug、测试体系起步
- **合并 0.4.2 的 UI 升级与表情包库**到本期，作为 V 类与 F4 的扩展

> v0.4.1 主题升级为「Stability + UX + UI」。**Week 5 起 feature freeze，只修 bug 不加功能**。

## 1. 不做的（明确划线）

- ❌ Snippets / 全局打字自动展开 / OCR / Smart Actions（留 v0.5.0）
- ❌ 新协议 / 新数据结构地震（保 v0.4.0 的兼容承诺）
- ❌ macOS 适配（本机不可 build，无意义）—— 注意 A7 是「为 Mac 除雷的样式兜底」，不等于做 Mac 适配
- ❌ 触碰内部兼容字段：`<!--TIEZ_RICH_IMAGE:`、`tiez-sync`、`tiez.log`、`tiez_xxx` localStorage、`tiez/tiez_xxx` MQTT topic
- ❌ Cargo `opt-level = "z"`（原 C7，桌面应用不该为体积牺牲性能；可考虑保守的 `lto = "fat"` + `strip`）

| **A8** | **迁移失败回滚 / 幂等性**：当前 v0.4.0 的 `com.tiez → app.magpie` 迁移没有「半失败」处理路径。修复：① 启动时检测 `app.magpie` 是否存在但为「半迁移状态」（有 db 但缺 settings 等）；② 迁移过程改为先复制到 `app.magpie.tmp`，复制完整后原子重命名为 `app.magpie`；③ 失败时弹中文提示并降级到使用旧目录 `com.tiez`，写明日志便于排查 | **P1** |
| **A9** | **应用内反馈入口**：当前设置面板「反馈」只是打开 GitHub Issues 页面，便携版用户报 bug 时还要自己从日志目录手动找 `tiez.log`。新增：① 「反馈」按钮旁加「复制诊断信息」按钮，一键把 `tiez.log` 最后 200 行 + 系统信息（Win 版本、应用版本、是否便携版、数据路径）+ 当前活跃设置摘要写入剪贴板；② **不上传任何东西**，只是减少用户复制日志的成本；③ 复制时按 v0.4.0 的脱敏规则过滤敏感字段（密码、token、URL 中的 query string） | **P1** |
| **A10** | **便携版「开机自启动」勾选不生效问题**（用户实际反馈，**P0**）。**根因**：① `toggle_autostart` 把 `current_exe()` 的绝对路径写入 `HKCU\...\Run\Magpie`，但便携版用户常常会移动便携版文件夹位置 / 把便携版放在外接 U 盘上，下次开机时旧路径已失效；② `main.rs` 同时启用了 `tauri_plugin_autostart` 插件和自定义 `toggle_autostart` 命令，两套机制并存，可能存在 `Run` 项被互相覆盖或注册到不同 key（插件默认用 productName）。修复方案：① **统一只用一套机制**——直接使用 `tauri_plugin_autostart` 提供的 `app.handle().autolaunch()` API，删除自定义 `toggle_autostart` / `is_autostart_enabled` 中的注册表直写代码，由插件统一管理；② **便携版健壮性**：插件写入的命令保留 `--minimized` 参数；启动时如果检测到「期望便携模式但 `exe_dir/data/` 不存在」（可能是用户挪了文件夹但保留了 Run 项），写日志、降级为标准模式启动并提示用户；③ 自启动注册时若发现旧 `Run\TieZ` / `Run\tie-z` 残留则一并清理（v0.4.0 已部分实现，迁移到插件路径下保持兼容）；④ 加 e2e 测试：便携版构建产物 → 启动 → 勾选自启动 → 检查 `Run\Magpie` 值 → 重启验证 | **P0**（用户实际反馈） |
| **A7** | **Mac 同步设置输入框灰色 / 不可填问题修复**（来自上游飞书反馈）。表象：Mac 上设置 → 云端同步的账号/密码/服务器输入框看起来像被禁用（灰色）无法输入。**已确认代码无显式 `disabled` 条件**，是 **Mac 系统默认表单样式覆盖了 CSS 变量**的视觉问题。修复：① 为 `.search-input`、`input[type="text"]`、`input[type="password"]` 显式设定 `background-color`、`color`、`-webkit-appearance: none`；② 复测覆盖所有 input 字段（不只同步面板）。**约束**：只做「明显正确」的 additive 样式兜底（对 Windows 零影响），不做复杂跨平台适配；本仓库无法验证，标注「未在 Mac 实机验证」 | **P3**（为未来 Mac 适配除雷，成本极小、Windows 零风险） |

## 5. C 类：性能 / 稳定性内功

| 编号 | 内容 | 优先级 |
|------|------|--------|
| C1 | **基准测试套件**：`benches/` 目录，criterion 测三个核心 op：`get_history(0, 100)`、`search('keyword', 1000条)`、`insert_clipboard()`。出 `perf-baseline.md` | P0 |
| C2 | **大列表实测**：mock 5000 / 20000 / 100000 条数据看 `VirtualClipboardList` 滚动 fps、内存占用、搜索响应 | P0 |
| C3 | **内存泄漏排查**：跑 24 小时持续监听剪贴板（脚本每秒 setText）。**验收：RSS 增长 < 50 MB**。重点查 `richTextSnapshot` 的缓存 Map 和 sensitive_align 队列 | P1 |
| C4 | **启动速度优化**：实测 cold start ms。可能优化点：① `init_db` 的 schema 检查走异步、② 主题应用前先显示窗口骨架、③ `start_services` 改并行 `tokio::join!`。**验收：比 v0.4.0 快 ≥ 30%** | P1 |
| C5 | **Win+V 注册健壮性 / 接管系统快捷键**：当用户在设置中选 `Win+V` 作主快捷键时，Magpie 当前会因系统级 Win+V 占用直接报「快捷键不可用」（v0.4.0 已知问题）。**底层 `trigger_registry_win_v_optimization` 已实现，缺 UI 开关与失败兜底**。修复：① 在 ClipboardSettingsGroup 恢复"让 Magpie 接管 Win+V"开关；② 注册失败时检测是否因系统占用，弹一次性提示「需要释放系统 Win+V，是否启用接管？」；③ 接管时调 `trigger_registry_win_v_optimization(true)` 写 `DisabledHotkeys` 并提示重启资源管理器；④ 关闭时还原；⑤ 同时检测 PowerToys / Ditto 占用 Win+V 的场景给出说明 | P0（v0.4.0 实际反馈）|
| C6 | **Panic 兜底**：① 全局 `panic_hook` 写到 `tiez.log`、② grep 一遍 `.unwrap()` 把可恢复的换 `.unwrap_or_else(...)`、③ 主线程 panic 时尝试优雅落盘 db | P1 |
| C8 | **CI 加 Rust cache**：`Swatinem/rust-cache@v2`，预期 CI 时间从 30+ 分钟 → 10~15 分钟 | P2（cosmetic） |
| **C9** | **测试体系起步**：① 至少为 F1（标签快速操作）、C5（Win+V 接管）、A10（便携版自启动）写 e2e 测试（playwright）；② 为剪贴板核心 op 写 vitest 单元测试（去空白、tag 关联）；③ 接到 GitHub Actions CI 跑；④ 不追求高覆盖率，只测「最容易回归的 happy path」 | **P1** |

> 删除原 C7（`opt-level = "z"`）。如需体积优化，在不损失性能的前提下尝试 `lto = "fat"` + `strip = "symbols"` 即可，不进 P0/P1 清单。

## 6. V 类：UI 升级（合并原 0.4.2 主项）

|------|------|--------|------|

| V2 | **Toast 提示统一**：复制成功 / 失败 / 网络错误的 toast 风格统一（颜色、图标、消失时长） | P2 | 抽一个 `<Toast>` 组件 |

## 10. 风险与回滚

| 风险 | 缓解 |
|------|------|
| C4 启动并行化导致 race condition | 严格用 `tokio::join!` 等待，不 fire-and-forget；加单元测试 |
| C6 panic_hook 写文件本身崩溃造成无限递归 | hook 内部用 `std::panic::catch_unwind` 兜底 |
| A2 卸载弹窗在静默卸载场景挡道 | 检测命令行参数 `/S`（NSIS 静默标志）时跳过弹窗，发 `WM_CLOSE` 走优雅关闭 |
| A10 删自定义 toggle_autostart 后老用户 Run 项遗留 | 启动时检测 `Run\Magpie` 旧值（绝对路径），如不匹配当前 exe 则用插件 API 重写一遍 |
| C5 接管 Win+V 写注册表后用户感觉被偷袭 | UI 必须显式开关 + 一次性弹窗确认 + CHANGELOG 显式声明 |
| F4 表情包占磁盘空间增长不受控 | 每个表情压缩到 < 100 KB；总数提示阈值（如 1000 个）；内置表情包总大小 < 5 MB |
| V5 卡片密度切换导致虚拟列表 item 高度计算混乱 | 切换时强制重算 `VirtualClipboardList` 的 itemHeight；加视觉测试 |
| V6 设置面板重组导致老用户找不到设置项 | 所有设置项 ID 不变，只是分组变；提供搜索框（已有）；首次升级时弹一次「设置面板已重组」提示 |
