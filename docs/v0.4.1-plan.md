# Magpie v0.4.1 计划：Stability + UX

> 状态：v0.4.0 发布后定稿
> 主题：稳定性 / 性能内功 + 上游 issue 借鉴 + UX 增强（合并原 0.4.2 计划）
> 周期：v0.4.0 发布后 3~4 周
> 路线图归属：方案 A（迁移巩固）+ 方案 C（性能/稳定性专项）+ U/F 系列上游借鉴

---

## 0. 不做的（明确划线）

- ❌ Snippets / 全局打字自动展开 / OCR / Smart Actions（留 v0.5.0）
- ❌ 大规模 UI 重设计（留 v1.0）
- ❌ 新协议 / 新数据结构地震（保 v0.4.0 的兼容承诺）
- ❌ macOS 适配（本机不可 build，无意义）
- ❌ 触碰内部兼容字段：`<!--TIEZ_RICH_IMAGE:`、`tiez-sync`、`tiez.log`、`tiez_xxx` localStorage、`tiez/tiez_xxx` MQTT topic

## 1. A 类：迁移巩固 / Hotfix（基于实际反馈）

| 编号 | 内容 | 优先级 |
|------|------|--------|
| A1 | v0.4.0 上线后**观察期收集 issue**，针对实际问题修 | P0（看反馈） |
| A2 | **卸载体验改善**：检测到 Magpie 进程时弹"是否关闭？"提示框，而非 `taskkill /F` 硬杀；NSIS 静默卸载（`/S`）时跳过弹窗 | P1 |
| A3 | **迁移日志可见性**：在 `tiez.log` 头部输出 `[MIGRATION v040] from=... to=... db_size=...`，便于用户/自查 | P1 |
| A4 | **更新检查错误分类**：失败时把网络层（DNS / TLS）错误分类，给出"GitHub 暂时不可达"等中文提示，不再直接抛英文 | P2 |
| A5 | **datapath.txt 健康检查**：首次启动若 datapath 指向的盘符不存在则回退默认目录并写日志 | P2 |
| **A6** | **替换设置 → 云端同步的飞书教程链接为本仓库 GitHub 教程**。当前 `SyncSettingsGroup.tsx` 与 `CloudSyncSettingsGroup.tsx` 中"查看教程"按钮分别指向上游作者飞书文档（`https://my.feishu.cn/docx/...`）。问题：① 上游私域，链接随时可能挂；② 国外用户访问飞书有困难。修复方案：① 在 `docs/cloud-sync-tutorial-mqtt.md` 与 `docs/cloud-sync-tutorial-webdav.md` 编写本仓库自己的教程并入 git 仓库；② 链接改为指向 `https://github.com/Duojiyi/magpie/blob/master/docs/cloud-sync-tutorial-{mqtt,webdav}.md`；③ **便携包打包脚本** `scripts/build-portable.ps1` 把这两份 md 一并塞进便携版（与 README/LICENSE 同级），便于离线场景查阅 | **P0**（用户明确要求 v0.4.1 修）|
| **A7** | **Mac 同步设置输入框灰色 / 不可填问题修复**（来自上游飞书反馈：[jimuzhe/tiez-clipboard 飞书 issue]）。表象：用户在 Mac 上打开设置 → 云端同步，账号/密码/服务器等输入框看起来像被禁用（灰色），无法输入。**已确认代码无显式 `disabled` 条件**（仅"立即同步"按钮在 syncing 时禁用），故不是逻辑禁用，而是**视觉/CSS 问题**：Mac 系统默认表单样式覆盖我们的 CSS 变量。修复方向：① 在 Mac 上为 `.search-input`、`input[type="text"]`、`input[type="password"]` 显式设定 `background-color`、`color`、`-webkit-appearance: none`；② 复测对所有 input 字段（不仅同步面板）；③ 加 Mac 平台 visual snapshot 测试到回归测试集 | **P1**（来自上游 Mac 用户反馈，本仓库不发 Mac 但留作未来 Mac 适配的基础修） |

## 2. U 类：上游 Bug 借鉴（已确认本仓库同样存在）

发版前扫描了上游 [jimuzhe/tiez-clipboard](https://github.com/jimuzhe/tiez-clipboard/issues) 的开放 issue，**已逐个比对当前代码确认未修**。本节 5 个 bug 全部纳入 v0.4.1 修复。

| 编号 | 内容 | 优先级 | 评估 |
|------|------|--------|------|
| U1（[#97](https://github.com/jimuzhe/tiez-clipboard/issues/97)）| **复制空格 / Tab 缩进时内容为空** — 代码复制场景高频出现，影响开发者用户体验 | P0 | 1~2 小时定位，应该是剪贴板捕获时去空白字符过激 |
| U2（[#102](https://github.com/jimuzhe/tiez-clipboard/issues/102)）| **重复内容合并后复制已打标签的内容标签消失** — 数据型 bug，标签是用户意图丢不得 | P0 | 中等，标签关联逻辑要复盘 |
| U3（[#1](https://github.com/jimuzhe/tiez-clipboard/issues/1)）| **鼠标滚轮穿透到其他应用** — 体验问题，固定窗口模式下显著 | P1 | 中等，需 Win32 钩子配合 |
| U4（[#74](https://github.com/jimuzhe/tiez-clipboard/issues/74)）| **多屏幕显示位置与图层问题** — 我们 v0.3.x 已加边角复位逻辑，可能仍有遗漏 | P2 | 复测当前行为后定 |
| U5（[#34](https://github.com/jimuzhe/tiez-clipboard/issues/34)）| **Windows 25H2 与微软输入法冲突吞字** | P3 | 大坑，IME hook 兼容性，看 25H2 用户反馈量再投入 |

> 不纳入的：上游 #86 / #103 已在 v0.3.5 cherry-pick；#94 是 macOS；#45 是 Linux；#106 / #105 与本仓库无关。

## 3. F 类：UX 增强（来自上游 feature 请求 + 自补）

合并原 v0.4.2 的小步快跑计划：从上游 feature 请求中挑出与 Magpie 调性契合、工作量可控的 4 项纳入。

| 编号 | 内容 | 优先级 | 设计要点 |
|------|------|--------|----------|
| F1（[#96](https://github.com/jimuzhe/tiez-clipboard/issues/96)）| **条目快速打标签**：选中条目按 `T` 弹快速标签输入框，即时回车保存。支持快速打"敏感"（[#85](https://github.com/jimuzhe/tiez-clipboard/issues/85)）等预置标签 | P0 | 复用 `tag_repo`，UI 在 `ClipboardItem` 上叠 floating 输入框；预置一个 `__sensitive__` 保留标签 |
| F2（[#95](https://github.com/jimuzhe/tiez-clipboard/issues/95)）| **数字快捷粘贴 `Ctrl+1~9`**：可选开关，启用后在主面板可见时按 `Ctrl+数字` 直接粘贴对应顺序条目 | P0 | 当前 `quick_paste_modifier` 只支持 modifier+点击，扩展为 modifier+数字键。**主面板隐藏时不监听**，避免污染浏览器/IDE 的标签切换快捷键 |
| F3（[#85](https://github.com/jimuzhe/tiez-clipboard/issues/85)）| **敏感内容快速标记**：选中条目按 `S`（或自定义快捷键）→ 一键打"敏感"标签；列表中对应条目用色块或图标突出 | P0 | 复用 F1 的标签机制，预置 `__sensitive__` 内置标签，UI 上视觉强调 |
| F4（[#100](https://github.com/jimuzhe/tiez-clipboard/issues/100)）| **图片快速添加到表情包**：图片条目右键菜单 → "添加到表情包"，进入用户表情库供日后复用 | P1 | 数据存 `%APPDATA%\app.magpie\emojis\`，新建 `user_emoji_repo`；与 `EmojiPanel` 复用渲染逻辑 |
| F5（[#98](https://github.com/jimuzhe/tiez-clipboard/issues/98)）| **全局 vs 应用内快捷键分离**：每个快捷键加 scope 字段（`Global` / `InAppOnly` / `BackgroundOnly`），InAppOnly 走 webview 内 `keydown`，不全局注册 | P1 | 设置面板每个 hotkey 加 scope 选择；改造 `register_hotkey` 路径分流；提供 v0.4.0 用户的无损迁移（默认全标 Global，行为与历史一致） |

> F5 是稍大的改造，但从用户反馈优先级看必须做。如果时间不够可拆 v0.5.0，但当前规划保留在 v0.4.1。

## 4. C 类：性能 / 稳定性内功

| 编号 | 内容 | 优先级 |
|------|------|--------|
| C1 | **基准测试套件**：`benches/` 目录，criterion 测三个核心 op：`get_history(0, 100)`、`search('keyword', 1000条)`、`insert_clipboard()`。出 `perf-baseline.md` | P0 |
| C2 | **大列表实测**：mock 5000 / 20000 / 100000 条数据看 `VirtualClipboardList` 滚动 fps、内存占用、搜索响应 | P0 |
| C3 | **内存泄漏排查**：跑 24 小时持续监听剪贴板（脚本每秒 setText），看 RSS 是否单调上涨。重点查 `richTextSnapshot` 的缓存 Map 和 sensitive_align 队列 | P1 |
| C4 | **启动速度优化**：实测 cold start ms。可能优化点：① `init_db` 的 schema 检查走异步、② 主题应用前先显示窗口骨架、③ `start_services` 改并行 `tokio::join!` | P1 |
| C5 | **Win+V 注册健壮性 / 接管系统快捷键**：当用户在设置中选 `Win+V` 作主快捷键时，Magpie 当前会因系统级 Win+V 占用直接报「快捷键不可用」（v0.4.0 已知问题）。修复：① 在 ClipboardSettingsGroup 恢复"让 Magpie 接管 Win+V"开关；② 注册失败时检测是否因系统占用，弹一次性提示「需要释放系统 Win+V，是否启用接管？」；③ 接管时调 `trigger_registry_win_v_optimization(true)` 写 `DisabledHotkeys` 并提示重启资源管理器；④ 关闭时还原；⑤ 同时检测 PowerToys / Ditto 占用 Win+V 的场景给出说明 | P0（v0.4.0 实际反馈）|
| C6 | **Panic 兜底**：① 全局 `panic_hook` 写到 `tiez.log`、② grep 一遍 `.unwrap()` 把可恢复的换 `.unwrap_or_else(...)`、③ 主线程 panic 时尝试优雅落盘 db | P1 |
| C7 | **Cargo opt 升级**：试 `opt-level = "z"` + `strip = "symbols"`，看体积能否再降。**前提**：跑回归测试确认没引入 bug | P2 |
| C8 | **CI 加 Rust cache**：`Swatinem/rust-cache@v2`，预期 CI 时间从 30+ 分钟 → 10~15 分钟 | P2（cosmetic） |

## 5. 推荐工作节奏（4 周）

```
Week 0:   v0.4.0 发布。开始 C1 + C2（性能基线）。

          ⑥ F2 数字快捷粘贴
          ⑦ U3 鼠标滚轮穿透
          ⑧ A7 Mac 同步输入框灰色（CSS 修）
Week 3:   ⑨ F5 快捷键 scope 分离
          ⑩ F4 图片加表情包
```

## 8. 风险与回滚

| 风险 | 缓解 |
|------|------|
| C7 `opt-level = "z"` 引入隐藏 bug | benchmarks + 手动冒烟，发现问题立即回 `"s"` |
| C4 启动并行化导致 race condition | 严格用 `tokio::join!` 等待，不 fire-and-forget；加单元测试 |
| C6 panic_hook 写文件本身崩溃造成无限递归 | hook 内部用 `std::panic::catch_unwind` 兜底 |
| A2 卸载弹窗在静默卸载场景挡道 | 检测命令行参数 `/S`（NSIS 静默标志）时跳过弹窗 |
| C5 接管 Win+V 写注册表后用户感觉被偷袭 | UI 必须显式开关 + 一次性弹窗确认 + CHANGELOG 显式声明 |
| F5 快捷键 scope 改造引入用户既有快捷键失效 | 默认全部标 Global 保持原行为；提供"恢复默认"按钮兜底 |
| F4 表情包占磁盘空间增长不受控 | 每个表情压缩到 < 100 KB；总数提示阈值（如 1000 个） |
